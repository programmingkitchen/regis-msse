# MSSE 642 - Syllabus

## Course Information

**Course Number:** MSSE 642  
**Course Title:** Software Assurance

### Course Description

This course provides a detailed explanation of software assurance practices, methods, and tools required throughout the software development lifecycle. Students will apply lifecycle knowledge in exploring common programming errors and evaluate common testing tools.

### Prerequisite Courses

- Basic understanding of programming languages

### Course Outcomes

Upon completion of this course, learners should be able to:

- Evaluate software design practices for creating secure software
- Select appropriate software processes for managing secure software
- Demonstrate how software assurance tools are used to manage secure software
- Detect and remediate common programming errors affecting security in developed software
- Resolve an ethical dilemma through the appraisal of alternatives

## Course Overview

Welcome to Software Assurance (MSSE642)! This course will expand your knowledge of the software development lifecycle (SDLC) and software engineering within the context of secure software development. Employers value reduced production costs and delivery delays as a result of building secure software. Likewise, customers expect that they will not have a loss of revenue and reputation due to a breach resulting from software vulnerabilities. This course is designed to ensure that both the software developer and information assurance professional can meet employers' and customers' expectation that secure software is delivered.

The course topics address both the IEC/IEEE software development lifecycle and ISC2 CSSLP domains of knowledge. A primary focus of the course will be an examination of common software vulnerabilities and defense, including a deep-dive into Identity Access Management (IAM) and securing web servers with TLS. Through several project components you will also explore some of the tools used for identifying flaws in software.
