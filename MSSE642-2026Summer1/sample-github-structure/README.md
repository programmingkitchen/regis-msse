# Sample Git Hub Structure

## Overview 

- When submitting assignments use the diretory structure below to organize your assignments. 
- You can include code in your repo, but I'll be grading from writeups in markdown.   Keep your code and your assignments separate. 
- Work in a branch each week. 
- When you are completed with the work for the week, submit a pull request to merge your branch into the main branch. 
- In the channel for the week where the assignment is due, notify me in Slack that you are submitting your assignment and post a link to each assignment (main branch only). 


## Code Directory Structure (root of the repo)

- See the example in "sample-github-structure," but your structure should start at the root of your repo.  

```bash
.
├── assignments
│   ├── images
│   │   └── sample-images.png
│   ├── research-presentation
│   │   └── research-presentation.md
│   ├── vibe-coding-assignments
│   │   ├── vibe-coding-1.md
│   │   ├── vibe-coding-2.md
│   │   └── vibe-coding-3.md
│   └── weekly-projects
│       ├── project-1-labsetup.md
│       ├── project-2-threat-analysis.md
│       ├── project-3-penlab-1.md
│       └── project-4-penlab-2.md

```
